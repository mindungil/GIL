# gil v0.1.0-alpha

First public alpha. Released 2026-04-27.

## What is gil

gil is an autonomous coding harness — a CLI agent that conducts a long,
exhaustive interview up front to extract every requirement, then runs to
completion without asking the operator again. The interview can take an
hour; the run can take days. The contract is: nothing interrupts the
agent mid-run except an objective stop condition.

Other coding CLIs ask mid-task or stop incomplete. gil asks everything
before starting and finishes once it starts.

## Highlights of v0.1.0-alpha

- **Long-form interview engine.** `core/interview` drives Sensing →
  Conversation → Confirm with `SlotFiller` + `Adversary` + `SelfAuditGate`,
  saturation-based stop. The frozen spec is locked with a SHA-256 hash and
  is the single source of truth for the run.
- **Anthropic-native run engine.** `core/runner.AgentLoop` issues real
  tool-use turns through the official Anthropic SDK and dispatches a
  9-tool suite (`bash`, `read_file`, `write_file`, `edit` 4-tier
  SEARCH/REPLACE, `apply_patch` Codex DSL, `memory_*` 6-file bank,
  `repomap` PageRank, `compact_now`, `exec` Recipe runner).
- **Stuck detection + recovery.** Five patterns (RepeatedAction,
  RepeatedObservation, RepeatedError, Monologue, PingPong, ContextWindow)
  with four recovery strategies (`ModelEscalate`, `AltToolOrder`,
  `ResetSection`, `AdversaryConsult`, `SubagentBranch`). Lifted from
  OpenHands / Cline / Goose with line-level attribution.
- **Cache-preserving compression.** Hermes-pattern Head/Middle/Tail with
  anti-thrashing, paired with Anthropic's `cache_control` on the system
  prompt and the last 3 messages. Multi-day runs preserve the prompt
  cache prefix across compactions.
- **Multi-provider.** Anthropic native; OpenAI Chat Completions adapter
  (pure stdlib) covers OpenAI, OpenRouter, and OpenAI-compatible
  endpoints (vLLM, Ollama). Validated against `qwen3.6-27b` on a
  user-provided vLLM endpoint — see `docs/dogfood/`.
- **Sandboxing.** bwrap (Linux), Seatbelt (macOS), Docker per-command
  exec, SSH per-command exec + rsync sync; Modal and Daytona cloud
  drivers wired (real deployment requires user credentials).
- **Shadow Git checkpoints.** A separate `.git` outside the workspace
  records every iteration; `gil restore <id> <step>` rolls back to any
  prior checkpoint without touching the user's own repo state.
- **Mission-control TUI.** Phase 14 4-pane redesign with shared palette,
  glyphs, sub-cell progress bars, and 9 slash commands
  (`/help /status /cost /clear /compact /model /agents /diff /quit`)
  using real SDK calls (no stubs).
- **Fresh-install onboarding.** XDG-standard layout, `gil init`
  scaffolding, `gil doctor` 5-group readiness check, file-based
  credential store (0600) with interactive picker, auto-spawn of `gild`
  on first daemon-needing command.
- **Distribution.** `scripts/install.sh` curl-pipeable installer,
  `gil update` with installer-method-aware dispatch (script / brew /
  manual), GoReleaser config (16 binary matrix, deb + rpm + brew
  formula), VS Code extension scaffold, Python Atropos environment
  adapter, OIDC bearer-token middleware.
- **In-session UX.** AGENTS.md / CLAUDE.md / `.cursor/rules` tree-walk
  discovery merged into the system prompt; MCP server registry (global
  + project TOML); persistent permissions (`always_allow` /
  `always_deny`); cost calculator with embedded model price catalog;
  `gil export` / `gil import` for sharing sessions; global
  `--output text|json` on every read command.

Phases 1–13 complete. 14 e2e tests green. Four binaries shipped:
`gil` (CLI), `gild` (daemon), `giltui` (TUI), `gilmcp` (MCP server).

## Install

```bash
# curl-pipe installer (Linux/macOS, x86_64 + arm64)
curl -fsSL https://raw.githubusercontent.com/mindungil/GIL/main/scripts/install.sh | bash

# go install (any platform with Go 1.22+)
go install github.com/mindungil/gil/cli/cmd/gil@v0.1.0-alpha
go install github.com/mindungil/gil/server/cmd/gild@v0.1.0-alpha

# from source
git clone https://github.com/mindungil/GIL.git
cd GIL
make build              # produces bin/{gil,gild,giltui,gilmcp}
sudo make install       # installs to /usr/local/bin
```

Homebrew formula and goreleaser deb/rpm artifacts ship with the tag;
publish path is documented in `docs/distribution.md`.

## Quick start

```bash
gil init                                # XDG dirs + auth login + config stub
gil auth login anthropic                # paste API key (stored 0600)
gil doctor                              # 5-group readiness check

ID=$(gil new --working-dir $(pwd) | awk '{print $3}')
gil interview $ID                       # long-form, saturation-based
gil spec freeze $ID                     # lock requirements
gil run $ID                             # autonomous; checkpoints continuously

# observe
gil events $ID --tail
gil watch $ID                           # single-pane progress monitor
giltui                                  # 4-pane mission-control TUI
```

## Provider support

| Provider     | Adapter             | Validation                                                   |
| ------------ | ------------------- | ------------------------------------------------------------ |
| `anthropic`  | native SDK          | reference target; primary dogfood path                       |
| `openai`     | Chat Completions    | adapter unit-tested; live validation pending user creds      |
| `openrouter` | OpenAI-compatible   | uses the OpenAI adapter; live validation pending             |
| `vllm`       | OpenAI-compatible   | validated against `qwen3.6-27b` on a user-provided endpoint  |

The qwen run is documented in `docs/dogfood/2026-04-27-first-real-run-qwen.md`
(adapter smoke) and `docs/dogfood/2026-04-27-second-run-end-to-end.md`
(full daemon loop). Cost catalog includes `qwen3.6-27b` priced at $0
because it served from a local vLLM deployment.

## Known limitations

- Pre-1.0. The frozen-spec proto, the on-disk session layout, and the
  CLI surface may change between alpha tags. Migrations will be
  best-effort, not guaranteed.
- The full live e2e (interview → run) has been validated against
  Anthropic models only. The qwen path has been validated at the
  adapter boundary and through the daemon with a mock provider; the
  combined live path is the next dogfood item.
- Modal / Daytona / OIDC / VS Code Marketplace / Atropos training all
  have shipping code paths but require user-provided credentials or
  publish access for live validation. The status list is in the README
  under "외부 자원이 필요한 잔여 항목".
- The milestone summarizer's fallback when the provider call fails is
  currently a logged error — Phase 17 will harden it to an empty-summary
  fallback so a network blip mid-milestone cannot poison the event log.
- The TUI requires an interactive PTY; coverage in CI relies on unit
  tests against `core/slash/handlers` and the in-app components, not a
  live terminal smoke.

gil lifts patterns and code from seven reference harnesses, with
line-level attribution in every commit message that imports their
shape:

- **OpenHands** — stuck detection patterns and recovery strategies.
- **Cline** — memory bank (6-file markdown), shadow-git checkpoint
  layout, VS Code extension scaffold shape.
- **Aider** — repomap PageRank, SEARCH/REPLACE 4-tier edit DSL.
- **Codex (OpenAI)** — `apply_patch` DSL, bwrap and Seatbelt sandbox
  configurations.
- **Hermes Agent (Nous Research)** — cache-preserving compression
  Head/Middle/Tail pattern, Recipe / multi-step compression, Atropos
  environment adapter, OIDC JWT decode pattern.
- **Goose (Block)** — MCP client subprocess pattern.
- **OpenCode (sst)** — last-wins permission glob evaluator, file-based
  `auth.json` credential store layout.
- **grpc-gateway** — HTTP/JSON gateway over the gRPC core.

Their work is the reason gil exists. The decision log in
`docs/research/2026-04-25-reference-harnesses-deep-dive.md` and
`docs/progress.md` records what was lifted, what was rewritten, and why.
